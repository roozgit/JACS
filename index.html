Features

Equipment management

Enter equipment data based on ISO-14224 and add as much data for the equipment as you see fit. You can define levels 3 to 9 of ISO-14224 taxonomy hierarchy. Also you can easily move the item from one taxonomy level to another. All suggested equipment classes in ISO-14224 are pre-defined to make adding equipments easier. You can add to these classes too.

Maintenance management

Maintenance recording features include: issuing and tracking work orders, defining custom workflows for different work categories(corrective, preventive, or predictive), requesting parts from warehouse for maintenance, and specify which subunit or component have been repaired based on ISO-14224

Failure management

Define failure on any taxonomy level and attach this definition to the corrective action undertaken in response. Helphul tools for FMEA (Failure Mode and Effects Analysis) and RCFA (Root Cause Failure Analysis) have been added. History of equipments can be visualized in different manners (calendar or chart) at any time. Work hours after latest maintenance routine for each equipment is calculated and displayed to the user.

Part management

Tell the software which part has been used in which equipment and how many. Set minimum and maximum required amount of the part. Track part history, requests and procurement records. Get a report of material usage in the whole plant.

Personnel management

There is no limitation on the number of users who can login to the software. Each one has a seperate file and even work-shifts can be defined(on-demand). You can easily see the list of jobs that has been refered to a specific user.

Other features

Pictures of equipments or maintenances can be seen in a gallery fashion. You can easily attach an infinite (well, not infinite) number of documents to a plant, section, equipment or maintenance item. Also, it is possible to create consumption logs for the equipments (on-demand).
